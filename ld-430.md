## 项目主要需求

### 项目概述

【纯后端服务】数字素材库与共享 API 服务。面向设计师和创意团队提供数字素材（图片/矢量/字体/模板）的上传、分类管理、搜索、标签和权限共享的 RESTful API。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个，每个贯穿后端全链路）

1. **素材（Asset）** — 字段：素材标题、描述、文件类型（Image/Vector/Font/Template/Video/Audio/3DModel）、文件格式（PNG/JPG/SVG/OTF/TTF/MP4/PSD等）、文件URL、缩略图URL、文件大小（bytes）、分辨率（宽×高）、标签列表、分类ID、上传者ID、许可类型（Free/CC0/CC_BY/CC_BY_SA/Commercial/Extended）、状态（Draft/Published/Archived/Flagged）、下载次数、浏览次数
   - 业务动作：上传素材、编辑信息、发布/下架
   - **贯穿后端**

2. **分类（Category）** — 字段：分类名称、父分类ID（支持多级）、分类图标、排序、描述
   - **贯穿后端**

3. **素材集/收藏夹（Collection）** — 字段：收藏夹名称、描述、创建者ID、封面图URL、素材ID列表、是否公开、协作成员ID列表
   - **贯穿后端**

4. **下载记录（DownloadRecord）** — 字段：素材ID、下载者ID、下载时间、下载用途（Personal/Commercial/Editorial）、许可协议版本
   - **贯穿后端**

5. **标签（Tag）** — 字段：标签名称、标签类别（Style/Color/Subject/Mood/Technical/Other）、使用次数
   - **贯穿后端**

6. **审核记录（ReviewRecord）** — 字段：素材ID、审核人ID、审核结果（Approved/Rejected/NeedsRevision）、审核意见、审核时间
   - **贯穿后端**

### 横切关注点（≥ 2 个）

1. **RBAC 权限控制**
   - 角色：Admin / Moderator / Uploader / Viewer
   - 触达文件层：DB role表 + RBAC中间件 + controller权限检查 + 类型定义
   - 特殊规则：Viewer只能浏览和下载免费素材，Commercial许可素材需额外权限

2. **审计日志**
   - 记录素材上传、审核、下载、许可变更
   - 触达文件层：DB audit_log表 + 日志中间件 + service埋点 + 类型定义

3. **接口限流**
   - 下载接口限流（防爬），上传接口限流
   - 触达文件层：限流中间件 + Redis

### 共享枚举/常量（≥ 2 组）

1. **素材文件类型 AssetType**：Image / Vector / Font / Template / Video / Audio / 3DModel
   - 定义：`backend/src/types/enums.ts`

2. **许可类型 LicenseType**：Free / CC0 / CC_BY / CC_BY_SA / Commercial / Extended

3. **素材状态 AssetStatus**：Draft / Published / Archived / Flagged

4. **审核结果 ReviewResult**：Approved / Rejected / NeedsRevision

### 后端中间件清单（≥ 2 个）

1. **`authMiddleware`** — JWT 鉴权
2. **`rbacMiddleware`** — 角色+许可权限校验
3. **`auditLogMiddleware`** — 审计日志
4. **`errorHandlerMiddleware`** — 统一异常
5. **`rateLimitMiddleware`** — 接口限流（下载接口每IP每分钟10次，上传每小时20次）
6. **`requestLoggerMiddleware`** — 请求日志
7. **`validationMiddleware`** — 参数+文件校验

### 技术要求

- **英文短名（用于 Docker Compose）**：`asset-library-api`
- **后端端口**：19310 / 3000
- **后端**：NestJS + TypeScript
- **数据库**：MongoDB 7（文档型更适合素材元数据）
- **ODM**：Mongoose
- **对象存储**：MinIO（S3兼容，本地Docker部署，存储素材文件）
- **缓存**：Redis 7（限流 + 热门素材缓存 + 搜索结果缓存）
- **认证**：JWT
- **API文档**：Swagger

### 部署要求

后端 + MongoDB + Redis + MinIO，Docker Compose 编排。
- 顶层 `name: asset-library-api`
- 不写 `version:` 字段
- MongoDB + Redis + MinIO 均使用命名卷
- MongoDB healthcheck + 后端 depends_on service_healthy
- MinIO 端口 9000 (API) + 9001 (Console)
- `.env` 必含：`COMPOSE_PROJECT_NAME`, `MONGO_URI`, `JWT_SECRET`, `REDIS_URL`, `MINIO_ENDPOINT`, `MINIO_ACCESS_KEY`, `MINIO_SECRET_KEY`, `MINIO_BUCKET`, `BACKEND_PORT`

### Git 要求

`git init` + `.gitignore`。
